# 微信公众号文章创作

## 定位

创作公众号文章内容，并为后续“文章配图”和“文章发布”节点准备清晰素材。默认输出可编辑 Markdown；当用户要求可直接复制到公众号编辑器时，再额外输出独立 `.html` 文件。

## 工作流

1. 理解选题：
   - 提炼用户目标、读者、场景、核心观点、发布目的和希望读者采取的行动。
   - 输入不足时先做合理假设，并在成稿前用一句话标注关键假设。
   - 保留用户给出的事实、数据、案例和品牌表述；不要编造来源、采访、引用或统计数字。
2. 匹配文章主题：
   - 从文章类型中选择主调：技术知识、深度观点、商业职场、方法清单、产品发布、活动复盘等。
   - 根据主调确定标题风格、段落密度、案例比例、解释深度和视觉气质。
   - 当用户要求公众号 HTML 或视觉主题时，读取 `references/themes-index.md`，只加载一个最匹配的主题文件。
3. 设计文章结构：
   - 先给出标题候选、摘要/导语和大纲，再写正文。
   - 正文使用适合手机阅读的短段落、清晰小标题、必要的列表和自然过渡。
   - 开头要快速进入冲突、问题、趋势或收益；结尾要给出总结、行动建议或引导。
4. 写作与润色：
   - 默认使用中文公众号语气：清楚、克制、有观点，避免空泛鸡汤和夸张营销腔。
   - 技术/知识文章重视概念拆解、例子和可执行步骤；商业/职场文章重视判断框架和取舍；观点文章重视论证链条。
   - 对用户已有草稿，优先保留原观点，只调整结构、表达、节奏和标题。
5. 准备配图 prompt：
   - 为封面图输出一条可交给 `$wechat-article-image` 的 prompt，包含主题、核心隐喻、标题融合方式、比例、风格和负面约束。
   - 为关键章节规划 1-3 张文内配图 prompt，说明每张图对应的段落、视觉隐喻、比例和用途。
   - 长标题要压缩成适合画面的短主标题，完整标题仍保留在文章标题中。
6. 可选生成公众号 HTML：
   - 只有当用户要求“可复制到公众号编辑器”“排版 HTML”或“发布前 HTML”时，生成独立 `.html` 文件。
   - 使用简单结构和内联样式，避免外部 CSS、脚本、iframe、表单、事件处理器、远程字体和复杂选择器。
   - 默认不要生成正文 `<a>` 标签；外部资料改写为“参考资料”文本块，保留标题和完整 URL。
   - 代码块不要依赖 `<pre>`、`<code>` 或 `white-space`。用带内联样式的段落/区块承载代码，换行写成 `<br>`，缩进和连续空格写成 `&nbsp;`。

## 配图 prompt 格式

为后续配图节点输出这个结构：

```text
用途：封面图 / 文内配图
文章标题：[完整标题]
画面标题：[适合画面内展示的短标题]
主题：[文章主题]
核心隐喻：[一句话说明画面如何解释观点]
构图：[封面用 2.35:1；文内图按段落需要选择 16:9 / 4:3 / 1:1]
标题融合：[屏幕 / 文档层 / 标签 / 面板 / 场景物体]
风格：[与文章主题匹配的视觉气质]
负面约束：无水印、无 Logo、无额外文字、无错别字、无随机 UI 标签
```

如果用户只要文章，不要配图方案，省略详细 prompt，只保留一行“建议配图方向”。

## 公众号 HTML 兼容规则

- 从浏览器复制渲染后的富文本，不要复制 HTML 源码。
- 样式保持内联且保守。
- 正文默认不要依赖 `<a>` 超链接，把外部资料做成“参考资料”文本块。
- 图片优先使用可发布的 HTTPS 地址。本地相对路径只适合本地预览。
- 不要在最终公众号图片地址里使用 `file://`、私有 IP、纯 `http`、非标准端口或本地相对路径。
- 图片标签保持简单：全宽、`height:auto`、可选小圆角、描述性的 `alt`。

## 输出

默认输出：

- 一篇可继续编辑的 Markdown 文章。
- 标题候选和推荐标题。
- 摘要/导语。
- 封面图 prompt 和必要的文内配图 prompt。

如果创建文件，除非用户指定其他位置，否则放在工作区 `outputs/` 目录。使用清晰文件名，例如：

```text
outputs/wechat-article.md
outputs/wechat-article.html
```

最终回复中链接产物文件，并说明下一步可使用 `$wechat-article-image` 生成图片，或使用 `$wechat-article-publish` 创建公众号草稿。
