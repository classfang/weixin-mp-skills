# 微信公众号文章配图

## 定位

为文章创作节点准备好的文章主题、标题、摘要、章节或配图 prompt 生成图片。默认覆盖两类产物：微信公众号封面图/头图，以及正文中解释观点的文内配图。

## 核心偏好

优先采用成熟、精致的中文公众号视觉风格：

- 图片优先，标题其次：插画或场景本身应承载核心观点。
- 避免只有标题的海报版式、大面积空白标题区和泛泛的主视觉文字。
- 将标题融入画面，例如做成 UI 标签、玻璃面板、屏幕、文档层、标牌、注释或场景中的物体。
- 除非用户指定其他比例，文章封面/头图使用 2.35:1。
- 偏好高级知识媒体审美：干净、有结构、有质感，手机端可读。
- 避免廉价赛博朋克杂乱感、塑料渐变、超大描边字、刺眼特效，以及单一紫色/蓝色调的单调画面。

## 图片类型

- 封面图/头图：默认 2.35:1，强调一眼识别文章观点，适合公众号列表和文章顶部。
- 文内配图：根据段落内容选择 16:9、4:3 或 1:1，强调解释概念、流程、对比或结构，不要塞满小字。
- 系列配图：保持同一视觉系统、光线、材质和色彩，只改变构图焦点与信息层级。

## 工作流

1. 读取文章标题、摘要、正文片段或来自 `$wechat-article-write` 的配图 prompt。
2. 提炼核心隐喻，而不只是照搬标题。
3. 选择能一眼解释观点的视觉系统：工作流、对比装置、机器、仪表盘、爆炸图、数据流、层叠卡片或编辑部式场景。
4. 构图应以画面为主。除非用户要求做海报，否则主标题占画面面积不超过 10%-15%。
5. 将标题嵌入场景中，并通过景深、重叠、连线、卡片或界面逻辑与主视觉关联。
6. 标题文案保持短而准确。若文章标题较长，使用精简主标题加短副标题。
7. 明确抑制额外文字、错别字、水印、Logo、品牌标记和随机 UI 标签。
8. 除非用户只要可复用提示词，否则使用内置 `image_gen` 工具生成图片。

## 提示词结构

生成封面图时使用这个结构：

```text
生成一张微信公众号文章封面图，超宽幅 2.35:1。

主题：[文章主题]
核心隐喻：[用一句话说明画面如何解释观点]

方向：重图轻标题。主视觉应占画面 85% 以上。不要依赖空白标题区。

场景：[具体视觉系统：AI 工作台 / 层叠文档 / 战略仪表盘 / 爆炸式工作流 / 对比机器 / 编辑部视觉隐喻]

标题融合：把标题放进[玻璃 UI 面板 / 屏幕 / 文档层 / 物体标签]，作为场景的一部分，并与视觉系统相连。标题约占画面 10%-15%，手机端可读。

精确标题文本：
[主标题]
[副标题]

字体：现代中文无衬线，干净的纯白或银白文字，只允许轻微柔光或阴影。不要夸张描边、塑料渐变标题或装饰海报字。

风格：高级中文科技知识插画，干净商业编辑质感，3D 玻璃拟态加精致信息图 UI，层次清楚，细节锐利。

配色：深海军蓝 / 石墨色 / 银白 / 科技青，少量暖金点缀。避免过暗画面、廉价赛博朋克杂乱感和单调紫蓝渐变。

质量约束：高分辨率，适合微信公众号封面，无水印，无 Logo，无人物肖像，无额外文字，无英文乱码，无中文错字。
```

生成文内配图时使用这个结构：

```text
生成一张微信公众号文章文内配图，[16:9 / 4:3 / 1:1]。

对应段落：[章节名或观点]
核心解释：[这张图帮助读者理解什么]
场景：[流程图式工作台 / 对比仪表盘 / 层叠文档 / 概念模型 / 编辑部插画]

构图：画面主体清晰，不放大段文字，不做海报标题；允许少量抽象图形、箭头、编号模块和 UI 卡片表达结构。

风格：与封面图保持同一视觉系统，干净、有层次、适合手机阅读。

质量约束：高分辨率，无水印，无 Logo，无随机文字，无错别字。
```

## 视觉方案

AI、效率或知识类文章：

- 使用半透明工作台、界面卡片、数据流、分析面板、神经节点、芯片、箭头和模块化模板。
- 展示机制：提取、对比、反向流动、迭代、评分或模板生成。
- 让标题作为中央面板或屏幕上的主标签出现，而不是贴在空白背景上。

商业、战略或职场类文章：

- 使用仪表盘、文档堆叠、决策地图、流程图、会议白板、KPI 面板和层叠文件。
- 语气克制、专业；除非用户要求，否则避免卡通商务人物。

教程、清单或方法类文章：

- 使用结构化卡片、编号模块、工具界面、流程泳道和清晰视觉层级。
- 避免用可读小字填满画面。细节用抽象线条和块面表达。

## 标题处理

优先：

- 主标题：2-6 个中文字符，或一个紧凑短语。
- 副标题：必要时用一个短语补充年份、类别或收益。
- 标题放在可见物体或界面层里。

避免：

- 把完整文章标题做成巨大的海报标题。
- 留出大块左侧/右侧空白只放文案。
- 多个随机 UI 标签。
- 在图片里塞入密集段落。

如果用户提供长标题，应为图片压缩标题，并在文章正文/标题中保留完整标题，例如：

- 完整标题：`反推提示词：2026年AI普通人最强的“开挂技巧”`
- 封面标题：`反推提示词`
- 副标题：`2026 AI开挂技巧`

## 偏好总结

除非用户要求不同方向，否则保留这些审美默认值：

- 把图片视为主要叙事层。标题应辅助画面，而不是压过画面。
- 通过面板、屏幕、卡片、标签或空间层次把标题融入构图，而不是放在预留空白区。
- 中文字体精致简洁：现代无衬线、纯色、克制光影、层级清楚。
- 构建完整视觉场景，让标题周围有足够画面密度，使构图像有意设计，而不是文字贴在插画旁边。
- 偏好高级编辑清晰度，而不是装饰性炫技：可控光线、干净结构、层次深度、克制强调色。

## 输出

如果生成图片，直接提供最终图片，并说明比例和关键构图选择。如果用户只要求提示词，输出可直接复制的提示词，不生成图片。如果用户计划进入发布节点，提醒封面图路径可作为 `$wechat-article-publish` 的 `--cover` 输入。
